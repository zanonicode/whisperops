This Backstage template provisions a fully governed **Dataset Whisperer** agent with a 4-field form. In under 90 seconds of ArgoCD sync time, it creates:

## Scaffolded Repo Layout

Each scaffolded agent repo has a two-level layout:

```
agent-{name}/
  catalog-info.yaml      # Backstage Component â€” stays at root; invisible to ArgoCD
  manifests/             # ArgoCD sync path â€” only K8s resources here
    namespace.yaml
    bucket.yaml
    service-account.yaml
    service-account-key.yaml
    iam-bindings.yaml
    modelconfig.yaml          # unified Vertex AI Gemini 2.5 Pro ModelConfig
    configmap-system-prompts.yaml   # planner.md + worker.md
    agent-planner.yaml
    agent-worker.yaml
    toolserver-sandbox.yaml
    sandbox.yaml
    kyverno-policy.yaml
    networkpolicy-allow-a2a.yaml
    chat-frontend.yaml
    ingress.yaml
    argocd-app.yaml
```

ArgoCD Application `spec.source.path` is `manifests`, so it never tries to apply
`catalog-info.yaml` as a K8s resource (which previously produced a CRD-not-found
OutOfSync error for every new scaffold).

## Form Fields

| Field | Required | Description |
|-------|----------|-------------|
| `agent_name` | Yes | Slug, `^[a-z][a-z0-9-]{2,28}[a-z0-9]$` |
| `description` | Yes | Free-text description (max 200 chars) |
| `dataset_id` | Yes | One of: `california-housing`, `online-retail-ii`, `spotify-tracks` |
| `budget_usd` | No | Max spend in USD (default `5.00`, format: `\d+\.\d{2}`) |

`base_domain` and `project_id` are hidden fields, sed-baked into `template.yaml`
by `_vm-bootstrap` from the live VM IP and project ID at deploy time. Operators
never type them.

## Model Configuration

Both agent roles (Planner, Worker) share a single `ModelConfig` CR named `model`:

- **Provider:** `GeminiVertexAI` (kagent `v1alpha2` Declarative)
- **Model:** `gemini-2.5-pro` (us-central1)
- **Auth:** `kagent-vertex-credentials` Secret, Reflector-replicated from `kagent-system`
- **ProjectID:** baked from `project_id` at scaffold time

The 2-agent architecture (LangGraph Supervisor pattern) replaced an earlier 3-agent
chain (planner â†’ analyst â†’ writer) on 2026-05-14. The worker now both computes
*and* composes the user-facing response in a single A2A hop, with an internal
`execute_python` retry loop (up to 3 attempts) for stderr recovery. See the
session memory at `.claude/storage/memory-2026-05-14-2agent-chart-pivot.md` for
the architectural rationale.

## Budget Enforcement

Each agent's `whisperops.io/budget-usd` annotation declares a maximum spend.
`budget-controller` polls Mimir's `/alerts` API for fired `action=killswitch`
alerts and scales the agent's Deployments to 0 when triggered. Spend math lives
as Mimir ruler recording rules (`whisperops_spend_usd:cumulative`) backed by the
Vertex AI Gemini 2.5 Pro pricing table. The operator can re-enable an agent by
updating the budget annotation and triggering an ArgoCD re-sync.
